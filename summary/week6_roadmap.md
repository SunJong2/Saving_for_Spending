# SavingApp 기능 현황 & 로드맵

## ✅ 완성 (백엔드 + 프론트 둘 다)

```
회원가입 / 로그인          signup.html, login.html (+ 입력 검증)
홈 진입 가드               토큰 유무(1차) + 401(2차)
목표 상세 보기             홈에서 GET /goals/current (저금통 진행률 바)
목표 생성                  add-goal.html
절약 기록 추가             add-saving.html (+ 축하 모달)
최근 기록 목록             홈에서 GET /savings
```

핵심 순환(목표 세우기 → 아끼기 → 저금통 차오름 → 달성 축하) 완결.

## 🔨 지금 만들 것 — 백엔드는 있는데 프론트가 없음

| 기능 | 백엔드 API (완성됨) | 만들 프론트 |
|------|--------------------|-------------|
| 히스토리 목록 | GET /goals/history | history.html |
| 목표별 기록 보기 | GET /goals/{goal_id}/savings | history에서 목표 클릭 시 |
| 목표 수정 | PATCH /goals/current | edit-goal.html |
| 목표 삭제 | DELETE /goals/current | 홈에 삭제 버튼 + 확인 |
| 기록 삭제 | DELETE /savings/{saving_id} | 홈 기록 목록에 삭제 버튼 |

→ 이 5개를 끝내면 **MVP의 모든 백엔드 기능에 프론트가 붙음**

## ⏳ 나중에 추가 (지금 말고)

### 부가 기능 (원래 명세의 "추가 기능")
- 퀵 버튼 — 자주 쓰는 절약 항목 원터치 등록
- Streak — 연속 절약 일수
- 환산 표시 — "치킨 N마리 참음"
- 통계 — 카테고리별 비중, 주간/월간 추이 (SQL GROUP BY)

### 논의 후 보류
- target_amount 수정 (달성 재판정 로직 동반)
- 마감 초과 연장 UX (GET /goals/current의 d_day 음수 활용)
- 축하 모달에 % 달성 표시 (data.progress 추가로 쉽게 가능)
- 이미지 첨부 (누끼/상태바 노출 문제 → 보류)
- 홈 기록에 메모 표시 (text-overflow 말줄임)

### 인프라 / 마무리
- 홈 레이아웃 디자인 다듬기 (돼지 이모지 대체, 여백, 그림자 등)
- 배포 (Render)
- PWA (manifest.json — 홈 화면 추가)

### README에만 기록 (구현 안 함)
- 비밀번호 재설정 (이메일 인증 인프라 필요)
- 소셜 로그인, 그룹 챌린지

## 권장 진행 순서

```
1. 지금 만들 것 5개 (프론트 미완성 기능들)   ← MVP 완성
2. 홈 레이아웃 디자인 다듬기
3. 배포 (Render)
4. 부가 기능 중 취사선택 (통계, streak 등)
5. PWA
```

이유: 기능을 다 붙여 MVP를 완성한 뒤 디자인을 다듬는 게, 화면이 계속 바뀌는 상황에서 디자인을 여러 번 고치는 낭비를 막음. 배포를 부가 기능보다 앞에 두는 건 "실제 URL로 접속되는 완성품"이 포트폴리오에서 부가 기능보다 임팩트가 크기 때문.
